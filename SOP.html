Library SOP About Repo

Purpose

This SOP explains how to add and maintain training modules on this GitHub Pages site.

Quick Start (5 steps)

  1. Open the repository on GitHub in your browser.
  2. Click modules/modules.json → Edit.
  3. Copy an existing object, paste it, and change the fields (title, description, youtubeId, tags, etc.).
  4. Commit the change directly to main (or open a PR if you prefer reviews).
  5. Refresh the site — changes go live within ~1 minute.

Required Fields (per module)

  • id: a short unique slug (e.g., "probing-fanuc-basics").
  • title: the display title.
  • description: one or two sentences.
  • youtubeId: the 11-character YouTube video ID.
  • duration: optional string like "6:42".
  • tags: array of short labels (e.g., ["probing","cnc","beginner"]).
  • sopUrl: optional link to a detailed SOP/doc (PDF, Google Doc, etc.).
  • docUrl: optional link to any supporting document.

Finding a YouTube ID

From a YouTube URL like https://www.youtube.com/watch?v=AbCdEfGhIjk, the ID is the part after v= — here it’s AbCdEfGhIjk.

Thumbnails

Thumbnails auto-load from YouTube at https://i.ytimg.com/vi/<youtubeId>/hqdefault.jpg.

Tags

Choose 3–5 descriptive tags. Tag options auto-populate based on what exists in modules.json.

Change Review (optional)

If you want approvals, make changes on a branch and open a Pull Request. Use GitHub’s code owners for auto-reviewers.

Troubleshooting

  • If a module doesn’t show up, verify JSON is valid (use Preview or any JSON linter).
  • If a video won’t play, double-check the youtubeId.
  • For PDFs in the repo, link with a relative path (e.g., docs/probing_sop.pdf).
